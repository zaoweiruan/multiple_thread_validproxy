| ERROR | 4 | 错误信息 |

## 三、接口设计

### 3.1 Logger.h 新增接口
```cpp
// 日志等级枚举
enum class LogLevel {
    TRACE = 0,
    DEBUG = 1,
    INFO = 2,
    WARN = 3,
    ERROR = 4
};

// 初始化 (带等级配置)
static void init(const std::string& logDir, const std::string& prefix, LogLevel fileLevel = LogLevel::DEBUG, LogLevel consoleLevel = LogLevel::INFO);

// 带等级的写入
static void write(const std::string& msg, LogLevel level = LogLevel::INFO);

// 设置全局等级
static void setLevel(LogLevel level);

// 设置分层等级
static void setFileLevel(LogLevel level);
static void setConsoleLevel(LogLevel level);
```

### 3.2 main.cpp 日志函数
```cpp
// 重载logInfo/logError - 支持指定日志等级
void logInfo(const std::string& msg, LogLevel level = LogLevel::INFO);
void logError(const std::string& msg, LogLevel level = LogLevel::ERROR);
```

## 六、注意事项与风险

### 6.1 注意事项
1. 保持向后兼容 - 现有日志调用无需修改
2. 默认日志等级为INFO - 保证生产环境信息足够
3. 文件日志默认DEBUG - 方便问题排查
4. 控制台日志默认INFO - 减少噪音

### 6.2 潜在风险
1. 配置文件变更需要重新加载
2. 日志等级字符串解析需要处理大小写
3. 需要确保线程安全 - 多线程并发写入

### 使用步骤

```json

    "enabled": true,
    "level": "INFO",
    "console_level": "INFO",
    "file_level": "DEBUG"
}
```
2. 运行程序，日志自动根据等级输出至`bin/log/`
