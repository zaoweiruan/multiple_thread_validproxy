## 1. 概述

本文档描述代理管理面板的用户界面设计方案，基于数据表格风格（A 方案），实现代理列表、订阅管理、详情查看及日志展示的综合界面。

> **修订说明**: 本版反映现有代码库 `ProxyListPanel.cpp` 的实际结构，调整字段名和组件以保持一致性。

## 3. 组件规范

### 3.1 订阅列表

**字段** (基于 `Subitem` 模型):
- 编号 (#)
- 名称
- 数量 (代理数量)
- 启用状态 (绿色=启用, 红色=禁用)
- 最近更新时间 (`updatetime`)

**交互**:
- 双击订阅行 -> 弹出编辑窗口
- 编辑字段: remarks、url、enabled、autoupdateinterval、updatetime

### 3.2 代理列表

**字段** (基于 `ProfileExItem` 模型):

| 字段 | 可排序 | 说明 | 实现状态 |
|------|--------|------|----------|
| # | 否 | 编号 | ✅ 已有 |
| Host | 是 | 主机地址, 点击排序 | ✅ 已有 |
| Type | 否 | 协议类型 (Socks5/HTTP) | ✅ 已有 |
| Delay | 是 | 延迟时间 (ms), 点击排序 | ✅ 已有 (`delay`) |
| Speed | 否 | 速度 | ✅ 已有 |
| consecutive_failures | 是 | 连续失败次数, 点击排序 | ⚠️ 需添加显示 |
| Remarks | 否 | 备注信息 | ⚠️ 需添加显示 |
| Message | 否 | 状态消息 | ⚠️ 需添加显示 |

**排序状态循环**: `None -> Asc -> Desc -> None` (当前实现)

**交互**:
- 双击代理行 -> 弹出详情窗口
- 右键点击 -> 弹出菜单:
  - 测试: 对选中代理进行延迟测试
  - 生成配置: 输出代理配置 (需扩展支持分享链接)

### 3.3 详情窗口

> **注意**: 当前代码中尚未实现详情窗口组件，需新增 `ProxyDetailPanel` 组件。

### 3.4 日志窗口

纵向排列的日志展示区域，显示所有运行日志。

### 3.5 搜索功能

顶部搜索框，需添加到工具栏。

## 4. 数据模型参考

### 4.1 Subitem (订阅模型)

```cpp
struct Subitem {
    std::string remarks;         // 备注
    std::string url;             // 订阅地址
    bool enabled;                // 是否启用
    std::string autoupdateinterval;  // 更新间隔
    std::string updatetime;      // 更新时间
};
```

### 4.2 ProfileExItem (代理模型)

```cpp
struct ProfileExItem {
    std::string host;            // 主机地址
    int port;                    // 端口
    int delay;                   // 延迟 (ms)
    int speed;                   // 速度
    int consecutive_failures;    // 连续失败次数
    std::string remarks;         // 备注
    std::string message;         // 状态消息
};
```
